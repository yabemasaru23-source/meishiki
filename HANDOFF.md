## 絶対に守るルール

### 1. `PROTECTED HEAD BLOCK` を編集しない

`index.html` の `<head>` に以下のコメントで囲まれたブロックがあります。

```html
<!-- ▼▼▼ PROTECTED HEAD BLOCK — 編集禁止 / DO NOT EDIT ▼▼▼ -->
...
<!-- ▲▲▲ PROTECTED HEAD BLOCK END ▲▲▲ -->
```

中身は GA4 タグ・meta description・OGP・JSON-LD 構造化データです。
ここを上書きするとアクセス解析とSEOが失われます。
**SEO や解析まわりの変更を依頼された場合のみ、この中を触ってよい**とします。

### 2. コード変更後は必ず回帰テストを通す

```bash
node tests/verify.js
```

`✓ 全項目一致` が出るまで作業を完了とみなさないでください。
これは実在する2件の鑑定結果と突き合わせたテストです（後述）。

### 3. コメント区切りを消さない

`index.html` の `<script>` 内は3ブロックに分かれています。

```js
/* ================= 定数 ================= */
/* ================= 暦計算 ================= */   ← 節入り・ユリウス通日
/* ================= 命式 ================= */     ← ここまでが純粋ロジック
/* ================= 描画 ================= */     ← DOM 依存
```

`tests/verify.js` は「定数」から「描画」直前までを `vm` で切り出して実行します。
**この区切りコメントを消すとテストが動かなくなります。**
DOM に触るコードは必ず「描画」より下に書いてください。

## 計算ロジックの要点

自力で書き直す前に、なぜこの実装になっているかを理解してください。

**日柱** — ユリウス通日から `(JDN+9)%10` / `(JDN+1)%12` で算出。
基準日 2000-01-01＝戊午日、1949-10-01＝甲子日 の2点で検算済み。

**節入り** — 万年暦データを持たず、太陽黄経（Meeus 低精度法）を二分探索して求め、
JST（UTC+9）に変換しています。これにより全年代に対応します。
月柱は暦月ではなく **節**（立春・啓蟄…）で切り替わります。

**年柱** — 立春が境界。1月〜2月上旬生まれは前年の干支になります。

**蔵干** — 節入りからの日数で 余気 / 中気 / 本気 を判定。
月柱だけでなく **全ての柱に同じ日数を適用**します（日本の四柱推命で一般的な手法）。
これが正解データと一致することを確認済みです。

**通変星** — 日干と対象干の五行関係（比和・我生・我剋・剋我・生我）と陰陽の同異で決まります。

**天中殺（空亡）** — 日柱基準と年柱基準の2つを出します。
一般に「その人の天中殺」と呼ばれるのは日柱基準です。
**中殺** は年柱基準の天中殺に月支・日支が含まれるかで判定します（生月中殺・生日中殺）。

## デザインの指針

**「墨に五行」** — 背景は墨色（`--ink: #14161a`）。
色は装飾に使わず、**五行そのものにだけ**割り当てます。

| 五行 | 変数 | 値 |
|---|---|---|
| 木 | `--moku` | `#4f9e6f` |
| 火 | `--ka` | `#d2453e` |
| 土 | `--do` | `#c9922e` |
| 金 | `--kin` | `#bfc5cd` |
| 水 | `--sui` | `#4272ab` |

水は本来「黒」ですが、墨地では視認できないため藍で表現しています。

書体は見出しが Zen Old Mincho、本文が Noto Sans JP。
**新しい色や書体を足さないでください。** 足すときは理由とともに提案を。

中央の五角形（相生・相剋図）がこのページの signature です。
実線＝相生、破線＝相剋。円の大きさが五行の個数、
**欠けている五行は破線の空円**として描かれます。ここは崩さないこと。

## 今後のロードマップ

### 第2段階 — 姓名判断エンジン

- 新字体 → 旧字体の変換辞書（**ここが最大の難所**。姓名判断は旧字体画数が正）
- 五格（天格・人格・地格・外格・総格）の画数計算
- 1〜81画の吉凶・運勢テキストのマスターデータ
- マスターデータは `data/*.json` に外出しし、`index.html` から fetch する

### 第3段階 — ハイブリッド相性鑑定

- 姓名判断の相性：人格・地格の画数の下一桁を五行に変換し、相生・相剋を判定
- 四柱推命の相性：互いの命式のバランス、守護神に該当するか
- 出力は点数ではなく「強み」「ぶつかりやすい点」「開運アクション」の3本立て

### 第4段階以降 — 検討事項

- タブ切り替え（基本性格 / 仕事運 / 恋愛運 / 健康運）で縦スクロールを減らす
- 大運・年運の算出
- Next.js + Firestore への移行（**必要になってから**。今はやらない）

## 用語の注意

大森さんの資料は **四柱推命**、矢部さんのアプリ出力は **算命学** の系統です。
命式の立て方は共通ですが、上に載せる解釈体系が異なります。
「天中殺」「中殺」は算命学の用語です。

DB 設計上は **通変星マスターと中殺マスターを分けて**おいてください。
四柱推命の命式算出を土台に、算命学レイヤーを上に載せる構造が破綻しません。
